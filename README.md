# realtime-minigame-mvp

회식 자리에서 같은 URL 하나로 여러 명이 접속해 닉네임만 입력하고 바로 즐길 수 있는 아주 단순한 실시간 미니게임 웹앱입니다.

## 실행 방법

1. Node.js 22 이상을 준비합니다.
2. 프로젝트 루트에서 아래 명령을 실행합니다.

```bash
npm start
```

3. 브라우저에서 `http://127.0.0.1:4310` 또는 `http://localhost:4310` 으로 접속합니다.

## LAN 테스트 방법

1. 서버를 실행한 기기의 사설 IP를 확인합니다.
   macOS 예시: `ipconfig getifaddr en0`
2. 같은 Wi-Fi에 연결된 다른 기기에서 `http://사설IP:4310` 으로 접속합니다.
3. 각 기기에서 닉네임만 입력하고 입장합니다.
4. 2명 이상 모이면 아무나 `게임 시작` 버튼을 눌러 전체 흐름을 시작합니다.

## 게임 진행 흐름

1. 홈에서 닉네임 입력 후 입장
2. 대기실에서 참가자 목록 확인
3. 2명 이상이면 누구나 게임 시작
4. 아래 5개 게임이 고정 순서로 자동 진행
   1. 컬러 스냅
   2. GO / STOP 탭
   3. 숫자 헌터
   4. 10초 멈춰
   5. 게이지 스톱
5. 각 게임은 `안내 3초 -> 연습 1회 -> 본게임 1회 -> 결과 5초` 순서로 진행
6. 라운드별 순위 점수를 누적해서 최종 우승자 결정
7. 마지막 화면에서 `새 게임 시작` 버튼으로 로비 상태로 초기화

## 테스트

```bash
npm run smoke
```

스모크 테스트는 아래 흐름을 자동 확인합니다.

- 2명 입장
- 로비 반영
- 게임 시작
- 5개 라운드 자동 진행
- 점수 집계
- 최종 결과
- 새 게임 시작 리셋

## 현재 버전의 한계점

1. 서버 메모리만 사용하므로 서버를 재시작하면 참가자와 점수가 모두 초기화됩니다.
2. 동기화는 polling 기반이라 반응성이 WebSocket 방식보다 약간 느립니다.
3. 새로고침 복원은 같은 탭의 간단한 세션 정보만 사용하며, 복잡한 복구 기능은 intentionally 넣지 않았습니다.
